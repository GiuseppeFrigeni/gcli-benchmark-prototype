# Code of Conduct

## Our Standard

This project aims to be a welcoming, low-anxiety place to collaborate on contributor tooling, evaluation infrastructure, and Gemini CLI quality work. We expect participants to be respectful, constructive, and generous with context.

## Expected Behavior

- Be kind, patient, and specific in code review and issue discussion.
- Critique ideas, implementations, and tradeoffs without attacking people.
- Share enough context that others can reproduce problems and follow decisions.
- Assume good intent, then clarify with evidence when something is wrong.

## Unacceptable Behavior

- Harassment, discrimination, or personal attacks
- Intimidation, dogpiling, or deliberately hostile review behavior
- Publishing private information without consent
- Repeatedly dismissing contributors in ways that make participation unsafe

## Scope

This code of conduct applies to repository discussions, issues, pull requests, review comments, and any other project-managed collaboration channels.

## Reporting

If you experience or witness behavior that violates this code of conduct, open a private maintainer contact request through the repository owner rather than escalating publicly. Reports will be handled as promptly and respectfully as possible.
