# Publish Hosted GitHub Issue Backlog

Labels: `documentation`, `good first task`

## Summary

The repo now carries copy-paste-ready issue packets, but the hosted GitHub issue tracker still needs to be populated from those packets when GitHub tooling is available.

## Acceptance Criteria

- Create one hosted GitHub issue for each numbered packet file in `docs/issue-packets/`.
- Preserve the packet titles and labels when publishing.
- Update the packet README or roadmap index with the hosted issue URLs.
- Keep the repo packets as the editable source of truth even after publication.
